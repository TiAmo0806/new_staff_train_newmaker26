# 串口协议_算法端

本文件面向本仓库的视觉/算法侧开发，重点说明主程序什么时候发包、什么时候等 ACK。

## 1. 算法侧输入命令

串口命令入口在：

- `SerialCommandSource::next()`

当前支持从电控收到这些命令：

- `ARRIVE_BEAN (0x10)` -> 转成主流程命令 `arrive_bean`
- `ARRIVE_DIGIT (0x11)` -> 转成主流程命令 `arrive_digit`
- `RESET (0x12)` -> 转成主流程命令 `reset`
- `PING (0x13)` -> 直接回复 `PONG`

行为规则：

- 收到 `ARRIVE_BEAN` / `ARRIVE_DIGIT` / `RESET` 后，算法端立即先回 `ACK`
- 收到 `PING` 后直接回 `PONG`

## 2. 算法侧输出包

### 2.1 第一阶段输出：`BEAN_BIND`

触发时机：

- 状态机收到 `arrive_bean`
- 豆子区识别完成
- `VisionMemory` 中 `P1/P2/P3` 已满足发送条件

发送函数：

- `Protocol::makeBeanBindPacket()`

发送内容：

- `P1/P2/P3` 的豆子类别
- 每个豆子对应的目标数字

### 2.2 第二阶段输出：`FINAL_TASK`

触发时机：

- 状态机收到 `arrive_digit`
- 数字区识别完成
- `TaskGenerator` 成功生成任务

发送函数：

- `Protocol::makeTaskPacket()`

发送内容：

- `task_count`
- 最多 3 条任务

### 2.3 辅助输出：`ACK` / `PONG` / `ERROR`

- `ACK`：确认收到电控发来的命令包
- `PONG`：回应 `PING`
- `ERROR`：错误上报，当前主流程可扩展

## 3. 算法侧 ACK 与重发规则

逻辑在：

- `SerialPort::write()`
- `SerialPort::waitForAck()`

规则：

- 若 `serial.ack_timeout_ms <= 0`，发送后不等 ACK
- 若大于 `0`，则发送后等待匹配的 `ACK`
- 超时后最多重发 `max_resend` 次
- `ACK` 和 `PONG` 自己不再等待 ACK

匹配条件：

- `cmd == ACK`
- payload 至少 2 字节
- `payload[0] == expected_cmd`
- `payload[1] == expected_seq`

## 4. 算法侧关心的 payload 含义

### `BEAN_BIND`

算法侧应保证：

- `count` 为有效绑定数量
- 后续固定输出 3 组
- 无效项补 `0,255,0`

### `FINAL_TASK`

算法侧应保证：

- `success` 表示任务是否生成成功
- `task_count` 为有效任务数
- 后续固定输出 3 组
- 无效项补 `0,0,0`

## 5. 与主流程的关系

串口协议不是独立业务，而是主流程最后一段：

```text
CommandSource
-> Detector
-> RoiParser
-> TaskStateMachine
-> VisionMemory
-> TaskGenerator
-> Protocol
-> SerialPort
```

因此：

- `Detector` 不应该知道串口协议
- `Protocol` 不应该知道识别模式
- `SerialPort` 不应该知道任务语义，只负责字节发送和 ACK

## 6. 调试建议

### 只看协议

使用：

```bash
./serial_protocol_demo --mock
./serial_protocol_demo --port /dev/ttyACM0 --baudrate 115200 --send-bean-bind
./serial_protocol_demo --port /dev/ttyUSB0 --baudrate 115200 --send-final-task
./serial_protocol_demo --port /dev/ttyACM0 --baudrate 115200 --listen
```

说明：

- `--mock` 不开真实串口，只本地打印协议
- 真实串口联调时应显式写 `--port` 和 `--baudrate`

### 看真实主流程

使用：

- `debug_image_real_serial`
- `real_robot`

并打开这些日志：

```yaml
debug:
  print_tx_hex: true
  print_rx_hex: true
  print_parsed_packet: true
```
