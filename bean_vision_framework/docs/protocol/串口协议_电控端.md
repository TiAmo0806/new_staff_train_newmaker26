# 串口协议_电控端

本文件面向电控/C板开发，重点说明电控应该发送什么、会收到什么。

## 1. 基本串口参数

- `115200 8N1`
- 无流控

## 2. 电控发送给算法端的命令

### 2.1 `ARRIVE_BEAN (0x10)`

用途：

- 通知算法“已到达豆子区，可以开始识别 P1/P2/P3”

payload：

- 空

预期算法响应：

- 收到后立即回 `ACK`
- 完成豆子区识别后发送 `BEAN_BIND`

### 2.2 `ARRIVE_DIGIT (0x11)`

用途：

- 通知算法“已到达数字区，可以开始识别 L4-L8”

payload：

- 空

预期算法响应：

- 收到后立即回 `ACK`
- 完成数字区识别和任务生成后发送 `FINAL_TASK`

### 2.3 `RESET (0x12)`

用途：

- 清空算法端当前状态和缓存

payload：

- 空

预期算法响应：

- 收到后立即回 `ACK`

### 2.4 `PING (0x13)`

用途：

- 检查链路是否存活

payload：

- 空

预期算法响应：

- 回 `PONG`

## 3. 电控接收自算法端的包

### 3.1 `BEAN_BIND (0x04)`

含义：

- 告诉电控：
  - `P1/P2/P3` 上识别到了什么豆子
  - 该豆子最终应该去哪个 `digit_x`

payload：

```text
count
pickup_1 bean_1 target_digit_1
pickup_2 bean_2 target_digit_2
pickup_3 bean_3 target_digit_3
```

电控收到后应：

1. 校验 CRC
2. 解析 `cmd/seq`
3. 记录绑定关系
4. 回 `ACK(acked_cmd=0x04, acked_seq=收到包的seq)`

### 3.2 `FINAL_TASK (0x02)`

含义：

- 告诉电控最终搬运任务顺序

payload：

```text
success
task_count
from_1 to_1 bean_1
from_2 to_2 bean_2
from_3 to_3 bean_3
```

电控收到后应：

1. 校验 CRC
2. 解析任务列表
3. 回 `ACK(acked_cmd=0x02, acked_seq=收到包的seq)`

### 3.3 `ERROR (0x03)`

含义：

- 算法端主动上报错误

payload：

```text
error_code
```

建议电控处理：

- 记录错误
- 回 `ACK`

### 3.4 `PONG (0x05)`

含义：

- 对 `PING` 的响应

payload：

- 空

## 4. 电控需要实现的最小规则

### 发包时

- 所有包都按统一帧格式发送
- `seq` 由发送端自己维护

### 收包时

- 先验帧头
- 再验 `length`
- 再验 CRC
- 有效后按 `cmd` 分发

### ACK 时

ACK payload 固定为：

```text
acked_cmd acked_seq
```

例子：

- 若收到 `FINAL_TASK`，其 `cmd=0x02 seq=7`
- 电控回复的 `ACK` payload 应为：

```text
0x02 0x07
```

## 5. 电控最关心的编码

### 取货位

- `1 = P1`
- `2 = P2`
- `3 = P3`

### 放置位

- `4 = L4`
- `5 = L5`
- `6 = L6`
- `7 = L7`
- `8 = L8`

### 豆子

- `0 = soybean`
- `1 = mung_bean`
- `2 = white_kidney_bean`

### 数字

- `1 = digit_1`
- `2 = digit_2`
- `3 = digit_3`
- `4 = digit_4`
- `5 = digit_5`

## 6. 联调建议

先用：

```bash
./serial_protocol_demo --port /dev/ttyACM0 --baudrate 115200 --send-bean-bind
./serial_protocol_demo --port /dev/ttyUSB0 --baudrate 115200 --send-final-task
./serial_protocol_demo --port /dev/ttyACM0 --baudrate 115200 --listen
```

说明：

- `/dev/ttyACM0` 常见于 USB CDC ACM 设备
- `/dev/ttyUSB0` 常见于 USB 转串口芯片

确认电控能：

- 正确验包
- 正确解析 payload
- 正确回 `ACK`
