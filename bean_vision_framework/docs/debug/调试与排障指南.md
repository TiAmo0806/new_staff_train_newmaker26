# 调试与排障指南

建议按流水线顺序逐层排查：

```text
1. 图像输入
2. YOLO 检测
3. ROI 解析
4. 状态机 / VisionMemory
5. 任务生成
6. 协议打包
7. 串口发送
```

## 1. 程序能否启动

正常启动应至少看到：

```text
Runtime mode: ...
[STATE] WAIT_BEAN_COMMAND
Detector backend: onnxruntime model=...
```

常见问题：

- `App config not found`
- `YOLO model not found`
- `Detector load failed`
- `Serial open failed`

## 2. 图像输入问题

### 图片模式

若路径错误，会看到：

```text
Failed to read image: ...
```

### 相机模式

优先确认：

- 相机能否打开
- 分辨率是否符合预期
- 曝光是否过暗或过曝

可先用：

```text
camera_preview_demo
```

做纯预览调试。

## 3. YOLO 检测问题

常见现象：

- `detections=0`
- ONNXRuntime 推理异常
- 输出维度不符合 `1 x C x N`

优先检查：

- 模型路径
- `best.onnx` 与 `best.onnx.data` 是否同目录
- `conf_threshold` 是否过高

## 4. ROI 解析问题

现象：

- 明明检测到了但没有任何绑定
- 绑定位置错误
- 某个 ROI 经常漏识别

处理方法：

1. 打开 `save_result_image`
2. 查看 `debug_output/*_result.png`
3. 对照检测框中心点和 ROI 框的位置
4. 只修改 `config/roi.yaml`

## 5. 状态机问题

正常命令顺序：

```text
arrive_bean ...
arrive_digit ...
```

若顺序错误，会出现：

- `[WARN] expected arrive_bean ...`
- `[WARN] expected arrive_digit ...`

需要时可用：

```text
reset
```

清空状态和缓存。

## 6. 串口与协议问题

重点检查：

- 串口设备名是否正确
- `mock` 是否配置正确
- 是否收到 `ACK`
- 是否触发重发

独立联调建议优先使用：

```text
serial_protocol_demo
```

## 7. 推荐调试路径

1. 先用 `debug_command_image`
2. 再用 `camera_preview_demo` 调 ROI
3. 再用 `debug_camera_mock` 看多帧稳定性
4. 再用 `serial_protocol_demo` 验证串口
5. 最后用 `debug_image_real_serial` 和 `real_robot`
