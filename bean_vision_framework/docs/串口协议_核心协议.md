# 串口协议_核心协议

本文件以当前代码实现为准，描述统一协议本体。

## 1. 串口参数

- `115200 8N1`
- 无流控

## 2. 帧格式

统一帧结构：

```text
0xA5 | cmd | length | seq | payload... | crc_l | crc_h
```

字段说明：

- `0xA5`：固定帧头
- `cmd`：命令字
- `length`：payload 长度，单位字节
- `seq`：包序号，发送端每发一包自增
- `crc_l crc_h`：`CRC16-MODBUS`，低字节在前

解析规则对应：

- `Protocol::parsePacket()`
- `Protocol::makePacket()`

## 3. 命令字

| cmd | 名称 | 方向 | 说明 |
|---|---|---|---|
| `0x01` | `VISION` | 算法 -> 外部 | 调试保留，当前主流程不默认发送 |
| `0x02` | `FINAL_TASK` | 算法 -> 电控 | 最终搬运任务 |
| `0x03` | `ERROR` | 算法 -> 电控 | 错误码上报 |
| `0x04` | `BEAN_BIND` | 算法 -> 电控 | 豆子与目标数字绑定 |
| `0x05` | `PONG` | 算法 -> 电控 | 对 `PING` 的响应 |
| `0x10` | `ARRIVE_BEAN` | 电控 -> 算法 | 到达豆子区 |
| `0x11` | `ARRIVE_DIGIT` | 电控 -> 算法 | 到达数字区 |
| `0x12` | `RESET` | 电控 -> 算法 | 重置当前任务状态 |
| `0x13` | `PING` | 电控 -> 算法 | 链路探活 |
| `0x14` | `ACK` | 双向 | 确认某个已收到的包 |

## 4. 编码表

### 取货位编码

- `P1 = 1`
- `P2 = 2`
- `P3 = 3`

### 放置位编码

- `L4 = 4`
- `L5 = 5`
- `L6 = 6`
- `L7 = 7`
- `L8 = 8`

### 豆子编码

- `soybean = 0`
- `mung_bean = 1`
- `white_kidney_bean = 2`
- 无效/未知 = `255`（仅 `BEAN_BIND` 占位时出现）

### 数字编码

- `digit_1 = 1`
- `digit_2 = 2`
- `digit_3 = 3`
- `digit_4 = 4`
- `digit_5 = 5`

## 5. Payload 定义

### 5.1 `BEAN_BIND (0x04)`

payload 结构：

```text
count
pickup_1 bean_1 target_digit_1
pickup_2 bean_2 target_digit_2
pickup_3 bean_3 target_digit_3
```

总长度固定为 `10` 字节：

- `payload[0] = count`
- 之后固定保留 3 组，每组 3 字节

无效项占位：

```text
pickup=0 bean=255 digit=0
```

### 5.2 `FINAL_TASK (0x02)`

payload 结构：

```text
success
task_count
from_1 to_1 bean_1
from_2 to_2 bean_2
from_3 to_3 bean_3
```

总长度固定为 `11` 字节：

- `payload[0] = success`
- `payload[1] = task_count`
- 之后固定保留 3 组任务，每组 3 字节

空任务占位：

```text
from=0 to=0 bean=0
```

### 5.3 `ACK (0x14)`

payload：

```text
acked_cmd acked_seq
```

含义：

- `acked_cmd`：确认的是哪个命令字
- `acked_seq`：确认的是哪个序号

### 5.4 `ERROR (0x03)`

payload：

```text
error_code
```

### 5.5 `PING (0x13)` / `PONG (0x05)`

- payload 为空

### 5.6 `VISION (0x01)`

调试保留包：

```text
success p1 p2 p3 l4 l5 l6 l7 l8
```

未识别位置填 `255`。

## 6. ACK 规则

当前代码行为：

- 算法侧发送 `BEAN_BIND` / `FINAL_TASK` / `ERROR` 时，可等待 `ACK`
- 算法侧收到：
  - `ARRIVE_BEAN`
  - `ARRIVE_DIGIT`
  - `RESET`
  后会回 `ACK`
- 算法侧收到 `PING` 后回 `PONG`
- `ACK` 和 `PONG` 本身不再等待 `ACK`

## 7. 代码对应

- 打包：`Protocol::makeBeanBindPacket()` / `makeTaskPacket()` / `makeAckPacket()`
- 解包：`Protocol::parsePacket()`
- 等待 ACK：`SerialPort::waitForAck()`
- 电控命令转主流程命令：`SerialCommandSource::next()`
