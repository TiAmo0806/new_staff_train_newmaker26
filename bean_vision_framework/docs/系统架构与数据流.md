# 系统架构与数据流

## 1. 主调用链

工程当前没有多线程、消息队列或异步回调，整体是一条同步流水线：

```text
输入 / 命令来源
-> Detector
-> ROI 解析
-> VisionMemory / TaskStateMachine
-> TaskGenerator
-> Protocol
-> SerialPort
```

## 2. 模块职责边界

| 模块 | 负责 | 不负责 |
|---|---|---|
| `InputManager` | 获取 `cv::Mat` 图像 | 检测、ROI、任务 |
| `BeanNumberDetector` | ONNXRuntime YOLO 推理，输出 `vector<Detection>` | ROI 分配、任务生成 |
| `RoiParser` | 将检测框中心映射到 P1-P3 / L4-L8 | 多帧缓存、串口 |
| `VisionMemory` | 分阶段缓存豆子区和数字区结果 | 推理、协议 |
| `TaskStateMachine` | 控制 `WAIT_BEAN -> WAIT_DIGIT -> DONE` 状态流转 | 检测实现、协议细节 |
| `TaskGenerator` | 根据豆子和数字箱生成搬运任务 | 串口发送 |
| `Protocol` | 业务结构到协议字节帧的转换 | 串口开关 |
| `SerialPort` | 发送/接收字节、ACK 等待与重发 | 业务语义 |

## 3. 配置加载链

入口：

```text
src/main.cpp
```

加载顺序：

```text
app.yaml
-> classes.yaml
-> roi.yaml
-> serial.yaml
```

关键点：

- 路径优先按配置文件所在目录解析
- `classes.yaml` 负责类别别名归一化
- `roi.yaml` 只维护固定位置矩形

## 4. 状态机流程

```text
WAIT_BEAN_COMMAND
-> SCAN_BEANS
-> SEND_BEAN_BIND
-> WAIT_DIGIT_COMMAND
-> SCAN_DIGITS
-> GENERATE_FINAL_TASK
-> SEND_FINAL_TASK
-> DONE
```

### 第一阶段：豆子区

- 接收 `arrive_bean`
- 扫描 P1/P2/P3
- 缓存豆子类别
- 发送一包 `BEAN_BIND`

### 第二阶段：数字区

- 接收 `arrive_digit`
- 扫描 L4-L8
- 结合第一阶段缓存生成搬运任务
- 发送一包 `FINAL_TASK`

## 5. 豆子到数字的规则

```text
soybean           -> digit_1
mung_bean         -> digit_2
white_kidney_bean -> digit_3
```

## 6. ROI 解析规则

- 豆子类只允许进入 P1/P2/P3
- 数字类只允许进入 L4-L8
- 通过检测框中心点判断是否落入 ROI
- 同一 ROI 内取置信度最高的目标

## 7. 多帧识别规则

`debug_camera_mock` 和 `real_robot` 可使用多帧稳定识别：

- 每次扫描连续采样若干帧
- 每帧独立执行检测和 ROI 解析
- 按 ROI 和类别投票
- 达到票数与平均置信度阈值后才确认该 ROI
