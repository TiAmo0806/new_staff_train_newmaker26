# 文件映射表

本文档用于说明当前工程各文件夹下文件的用途。

说明规则：

- 自维护源码、配置、文档按文件逐项说明。
- 大量重复的样本图、构建产物、第三方头文件按模式归类说明。
- `build/`、`build_mingw/`、`debug_output/` 属于生成产物，不建议手改。

## 1. 根目录

| 文件 | 功能 |
|---|---|
| `.clangd` | 编辑器/语言服务器配置。 |
| `.gitignore` | Git 忽略规则，主要忽略 `build/`、`build_mingw/`、`debug_output/`、编辑器缓存和编译产物，同时保留 `config/`、`docs/`、源码目录以及 `bisai/` 下需入库的 ONNX Runtime 文件。 |
| `CLAUDE.md` | AI/工具协作说明。 |
| `CMakeLists.txt` | 主工程 CMake 构建入口，定义主程序和 demo。 |
| `README.md` | 项目入口说明和文档索引。 |
| `scripts/run_demo.sh` | 运行 demo 的辅助脚本。 |
| `src001_train_000003.png` | 本地测试图片样本。 |
| `src009_train_000016.png` | 本地测试图片样本。 |

## 5. `src/`

## 11. 建议手动维护的范围

建议日常只手动维护这些目录：

- `config/`
- `docs/`
- `include/`
- `src/`
- `tools/`
- 根目录入口文件

一般不要手动维护这些目录中的生成产物：

- `build/`
- `build_mingw/`
- `debug_output/`
- `bisai/build/`
- `bisai/output/`
- `bisai/output_verified/`
- `bisai/frames/`
