# 运行模式与主程序说明

## 1. 设计原则

本工程只维护一套主流程，不复制多套 `main.cpp`：

```text
命令来源 / 输入来源
  -> InputManager
  -> BeanNumberDetector
  -> RoiParser
  -> TaskStateMachine
  -> VisionMemory
  -> TaskGenerator
  -> Protocol
  -> SerialPort
```

运行模式由 YAML 配置切换。

主程序帮助：

```bash
./bean_vision_framework --help
```

会显示：

- 配置文件启动方式
- 四个运行模式的用途
- 命令驱动模式下支持的终端命令

## 2. 四个主程序模式

### 2.1 `debug_command_image`

定位：

```text
图片输入 + 终端命令 + mock 串口
```

用途：

- 验证 ONNXRuntime 模型加载
- 验证图片检测、ROI 映射、状态机、VisionMemory、任务生成、协议打包

命令：

```text
arrive_bean <image_path>
arrive_digit <image_path>
reset
quit
```

配置文件：

```text
config/debug_command_image.yaml
```

### 2.2 `debug_camera_mock`

定位：

```text
工业相机输入 + 终端命令 + mock 串口
```

用途：

- 调试工业相机曝光、增益、ROI 覆盖位置
- 进行 5 帧投票稳定识别

命令：

```text
arrive_bean
arrive_digit
reset
quit
```

关键规则：

- 每次扫描抓取 `frames_per_scan` 帧
- ROI 内同类出现次数 `>= min_vote_count`
- 平均置信度 `>= min_avg_confidence`
- 失败最多重试 `max_retry`

配置文件：

```text
config/debug_camera_mock.yaml
```

### 2.3 `debug_image_real_serial`

定位：

```text
图片输入 + 真实串口
```

用途：

- 不接相机时验证完整业务结果能否通过真实串口发给 C 板
- 验证 `BEAN_BIND` / `FINAL_TASK` / ACK / 重发

命令：

```text
arrive_bean <image_path>
arrive_digit <image_path>
reset
quit
```

配置文件：

```text
config/debug_image_real_serial.yaml
```

### 2.4 `real_robot`

定位：

```text
工业相机 + 真实串口 + C 板命令
```

用途：

- 整车运行模式
- 串口接收 `ARRIVE_BEAN` / `ARRIVE_DIGIT` / `RESET` / `PING`
- 串口发送 `BEAN_BIND` / `FINAL_TASK` / `ACK` / `PONG`

配置文件：

```text
config/real_robot.yaml
```

## 3. 两个独立 demo

### 3.1 `serial_protocol_demo`

职责：

- 单独测试串口与协议
- 不接相机
- 不跑 YOLO
- 不走完整状态机

典型命令：

```bash
./serial_protocol_demo --mock
./serial_protocol_demo --help
./serial_protocol_demo --port /dev/ttyACM0 --baudrate 115200 --send-bean-bind
./serial_protocol_demo --port /dev/ttyUSB0 --baudrate 115200 --send-final-task
./serial_protocol_demo --port /dev/ttyACM0 --baudrate 115200 --listen
./serial_protocol_demo --port /dev/ttyACM0 --baudrate 115200 --send-bean-bind --send-final-task --listen
```

说明：

- `--help` 会打印参数说明、设备名说明和默认行为
- 如果只写 `--port` 和 `--baudrate`，默认会发送一包 `BEAN_BIND`、一包 `FINAL_TASK`，然后进入监听

### 3.2 `camera_preview_demo`

职责：

- 单独测试工业相机预览
- 叠加 `roi.yaml` 中的 P1/P2/P3/L4-L8
- 保存当前帧
- 热重载 ROI

配置文件：

```text
../config/camera_preview_demo.yaml
```

帮助：

```bash
./camera_preview_demo --help
```

会显示：

- 配置文件启动方式
- `s` / `r` / `q` 按键说明

按键：

```text
s 保存当前帧
r 重新加载 roi.yaml
q 退出
```
